# ☕ 커피 생산관리 시스템 (Wireframe)

커피 생산공장의 생산관리 프로그램. 발주량을 입력하면 제품별 **라우팅 + 수율 + BOM**에 따라
**공정별 생산계획**과 **공정별 세부 작업지시서**를 화면에 표시하고 **인쇄**까지 할 수 있다.

> 현재는 **메뉴 구조를 연결하는 와이어프레임 단계**입니다.
> DB 연결과 실데이터 입력은 추후 단계이며, 지금은 HTML + 바닐라 JS + 정적 샘플데이터로 동작합니다.

## 실행

빌드 도구·서버·설치 없이 `index.html`을 브라우저로 열면 됩니다. (`file://` 더블클릭 가능)

로컬 서버로 보려면:

```bash
python -m http.server   # 그 후 http://localhost:8000 접속
```

## 배포 (Vercel)

빌드 과정이 없는 정적 사이트이므로 Vercel에 **무설정**으로 배포됩니다. GitHub 저장소를
Vercel에 연결하면 `main` 브랜치에 푸시할 때마다 자동 배포됩니다. Framework Preset은
`Other`, Build Command·Output Directory는 비워두면 됩니다(루트의 `index.html`을 그대로 서빙).

## 주요 흐름

```
발주 입력 / 발주 현황
        ↓
생산계획 산출  (발주량 → 라우팅·수율 역산 → 공정별 투입/산출/로스 + BOM 전개)
        ↓
공정별 생산계획
        ↓
공정별 작업지시서  → 🖨️ 인쇄
```

## 메뉴 구성

- **현황** — 대시보드
- **기준정보** — 제품 관리 / BOM 관리 / 라우팅·수율 관리 / 공정 관리
- **발주·주문** — 발주 입력 / 발주 현황
- **생산계획** — 생산계획 산출 / 공정별 생산계획
- **작업지시** — 공정별 작업지시 (인쇄)

## 구조

```
index.html              앱 셸 (상단바 + 좌측메뉴 + 본문)
assets/
  css/style.css         화면 스타일
  css/print.css         인쇄 전용 (작업지시서 1공정 = 1페이지)
  js/
    util.js   (U)       포맷/이스케이프
    data.js   (DB,MENU) 샘플 기준정보 + 메뉴 구조
    calc.js   (CALC)    생산계획 산출 엔진 (라우팅 역산·수율·BOM 전개)
    pages.js  (PAGES)   화면별 렌더링
    router.js (ROUTER)  해시 기반 라우터
    app.js              부트스트랩
```

자세한 아키텍처와 개발 규칙은 [CLAUDE.md](CLAUDE.md) 참고.
